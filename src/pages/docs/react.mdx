# Reactning Ichki Mexanizmlari

Bu bo‘lim React haqida asosiy bilimlarga ega bo‘lgan, lekin texnologiyaning ichki mexanizmlarini chuqurroq tushunishni istagan dasturchilar uchun mo‘ljallangan. Agar siz hali React bilan tanishmasangiz yoki asosiy qo‘llanmalarga muhtoj bo‘lsangiz, dastlab rasmiy React dokumentatsiyasini ko'rib chiqishingiz mumkin. Biroq, agar siz Reactning qanday ishlashini, uning qanday qilib kuchli bo‘lganligini va nima uchun bu texnologiya zamonaviy web ilovalar yaratishda asosiy vositaga aylanganini bilishni istasangiz, to‘g‘ri joydasiz.

Bu bo‘limda siz bilan React konseptlari bo‘ylab sayohat qilamiz va uning ishlash prinsiplarini tushunamiz. Biz Virtual DOM, Reconciliation, JSX, hooklar va Reactning boshqa asosiy tamoyillarini chuqur tahlil qilamiz. Shuningdek, Reactning kompilyator nazariyasi, fiberlar, va uning concurrent dasturlash modelini batafsil o'rganamiz. Bu bilimlar sizga React ilovalarini yanada samarali yaratishga yordam beradi.

Ushbu bo‘limda siz quyidagi mavzularni ko'rib chiqishingiz mumkin:

- **Virtual DOM**: Nega React Virtual DOMdan foydalanadi va bu qanday ishlaydi?
- **Reconciliation**: Reconciliation jarayoni qanday amalga oshiriladi?
- **JSX**: JSX kompilyatsiyasi va uning ishlash mexanizmi.
- **Fiberlar**: Reactning fiber arxitekturasi va uning concurrent dasturlash modeliga ta’siri.
- **React.memo va useTransition**: Bu kuchli primitivlar qanday ishlaydi va qanday holatlarda ishlatiladi?
- **Server rendering**: Server renderingni nima afzallik va kamchilik tomonlari bor va React server componentlar(RSCs)

Bu bo‘limda Reactning ilg'or(advanced) tushunchalarini o‘rganish orqali, siz nafaqat React ilovalarini yaratishda, balki ularni samarali ravishda optimallashtirishda ham ko‘proq bilim va tajribaga ega bo‘lasiz. Keling, sayohatimizni boshlaylik!


## Nega React Paydo Bo‘ldi?
Reactning paydo bo'lish sabab bo‘lgan asosiy omil – bu veb sahifalarning **yangilanishi**. Ilgari veb sahifalar asosan statik bo'lgan. Biz unda formalarni to'ldirardik, "Yuborish"(Submit) tugmasini bosardik, va butunlay yangi sahifani yuklardik. Bu usul ma'lum muddat yaxshi ishladi, lekin vaqt o'tishi foydalanuvchilarning talablari o'zgarib, yangi imkoniyatlarni paydo bo'lishiga turtki bo'ldi. 

Endi foydalanuvchilar yangi sahifalarni darhol yangilanishini, yangi sahifani kutmasdan natijalarni ko'rishni, veb sahifalarni yanada "jonli" va tezkor bo'lishini xohladilar. Bu esa o'z navbatida foydalanuvchi tajribasi(User experience) va foydalanuvchi interfeysi(User interface)ni yanada takomillashtirilishga sabab bo'ldi. Biroq, bu tezkor yangilanishlarni birdaniga keng ko'lamda amalga oshirish bir necha sabablarga ko'ra oson emas edi:

<Steps>
### Ishlash samaradorligi(Performance)
 Veb sahifalarini yangilash jarayonida ko‘pincha ishlash samaradorligi pasayib ketardi. Sababi, bizning o‘zgarishlarimiz brauzerni sahifa tuzilishini qayta hisoblash(reflow deb ataladigan jarayon) va qayta chizishga majbur qilardi, bu esa kechikishlarga olib kelardi.

 
### Ishonchlilik
 Katta va murakkab sahifalarda state(ma'lumot)ni izchil saqlab qolish qiyin edi. Bir nechta joyda stateni kuzatib borish va ularning barchasini sinxronlashtirish oson emas edi, ayniqsa, bir nechta dasturchilar bir kod bazasida ishlayotgan bo‘lsalar.

### Xavfsizlik
 Sahifaga kiritilayotgan HTML va JavaScript kodlarini xavfsiz qilish, cross-site scripting(XSS) va cross-site request forgery(CSRF) kabi xavflarni oldini olish har doim dolzarb masala edi.
</Steps>

React aynan mana shu muammolarni hal qilish uchun yaratilgan. Ushbu muammolarni yaxshiroq tushunish va Reactning qanday qilib ularni hal qilishini anglash uchun, Reactning paydo bo‘lishi va uni yaratishga olib kelgan omillarni bilishimiz kerak. Keling, React paydo bo'lishidan oldingi jarayonni o‘rganishga kirishamiz.
